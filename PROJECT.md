# GF2 点击助手 — 项目说明

面向《少女前线 2：追放》咖啡厅订单的**外部辅助**：用 **OpenCV 在截图里找图标**判断订单里出现哪些配料，再按规则点击；**主配料/补充**的落点来自 `points.json`，**提交**的落点来自对「提交」模板的匹配位置。不读取、不修改游戏进程内存，不注入 DLL。

---

## 项目概括

### 识别和点击是两套数据（不要混用）

| 环节 | 数据从哪来 | 说明 |
|------|------------|------|
| **识别「有没有这个配料」** | `click_shots/` 里的 **PNG/JPG 等模板图** | 对截图切出的 **配料 ROI**（见下）做 `matchTemplate`；**与 JSON 里该配料的坐标无关**。匹配到的 `(mx,my)` 在代码里**只用于**候选从左到右排序、以及 `seen_active` 去重 key，**不作为点击坐标**。 |
| **点主配料 / 点补充** | `calib/points.json`（及锚点）里的 **`abs_x` / `abs_y`（及 `rel_*`）** | 经 `build_target_points` 与窗口偏移、GUI「锚点修正」、紫色水果整体微调等换算后的**固定槽位/按钮坐标**。**不是**「在 JSON 坐标处做识别」。 |
| **点提交** | **不读 JSON** | 对**整幅**窗口灰度图用 `find_best_match(提交1 模板)`，点击**匹配到的矩形中心**（`gf2_bot.py` 中 `frame_left + sx + width/2` 等）。`points.json` 里通常也没有提交点。 |
| **锚点** | JSON 里「锚点1」的坐标；可选 `锚点1` 模板图 | 默认 `USE_WINDOW_TOPLEFT_AS_ANCHOR=True`：用**窗口客户区左上角**与 JSON 锚点坐标算偏移，**不**用锚点图匹配。若关闭该开关，会在**全图**上匹配 `锚点1` 模板以算偏移（仍是为点击坐标服务，不是识别配料订单）。 |

**配料识别 ROI（与代码一致）**：`gf2_bot.get_roi_rect` 为窗口**右上角**宽 `ROI_WIDTH`（默认 380）× 高 `ROI_HEIGHT`（默认 200）的矩形。主配料模板**只在此 ROI 内**做 `matchTemplate`，用于判断「订单预览里是否出现该配料小图标」。

**识别位置与点击位置是两块屏上区域（设计如此）**：例如在 ROI 里匹配到 `黄油1.png` 表示「本单需要黄油」；随后程序点击的是 `points.json` 里黄油槽位的坐标（如标定 **795, 790**）。小图在右上、槽位在下方/左侧等**完全可以不同**，**不要求**槽位像素落在 ROI 里。此前若把「槽位 abs_x 是否在 ROI 内」当成识别条件，属于对流程的误解。

同名对应关系：`click_shots/水果4.png` 等与 JSON 里同名点位；**先**在 ROI 内匹配到「订单里出现了该配料」，**再**点击 `target_points["水果4"]` 等（来自 JSON 换算）；若配置了 `AUTO_REPLENISH` 再点对应「补充*」。

### 一览

| 项目 | 说明 |
|------|------|
| **用途** | 在配料 ROI 内匹配主配料模板，按规则点击 JSON 中的槽位与补充；空闲后再在全图上匹配「提交」模板并点击其匹配中心。 |
| **识别** | 主配料：**ROI** + `click_shots/` + OpenCV。提交：**整幅窗口灰度图** + `提交1` 模板。 |
| **点击坐标** | 主配料与补充：**来自** `points.json`（经锚点/GUI 等换算），与匹配位置无绑定。提交：**来自**提交模板匹配框中心，**不**用 JSON。 |
| **入口** | `gf2_gui.py`（GUI）、`gf2_bot.py`（命令行）；核心逻辑在 `gf2_bot.py`。 |
| **权限** | 游戏若以管理员运行，本工具也需管理员权限才能向该窗口投递点击（打包 exe 可带 UAC 提权）。 |

**对齐客户区**：通过 Windows 窗口 API 调整游戏窗口**客户区**尺寸，与标定所用分辨率对齐；属于系统层窗口几何操作，**不是**改内存、也不是改游戏内渲染设置。

## 运行模式（GUI 下拉框）

三种模式由 `gf2_bot.run_bot(..., run_mode=...)` 控制，默认 **普通模式**。

### 1. 普通模式（`normal`）

- 在订单 ROI 内用 OpenCV 匹配到某个**主配料**模板（`click_shots/`）后，再点击 `points.json` 里该配料的**点击坐标**。
- 若该配料在配置里配置了「自动补充」映射（`AUTO_REPLENISH`），则**再点击一次**对应的「补充*」按钮坐标（如「补充水果1」「补充黄油1」等）。
- 一段时间无新识别后，在**整幅窗口截图**上匹配 `提交1` 模板，命中则点击**匹配位置中心**（非 JSON 坐标）。
- 行为即：主配料点击 + **全部适用的补充点击** + 自动提交逻辑（与历史版本一致）。

### 2. 无限材料模式（`infinite_material`）

- 与普通模式相同：仍会识别并点击**主配料**、仍会尝试**自动提交**。
- **区别**：命中主配料后，**一律不再点击**任何「补充*」按钮（包含补充水果、补充饮料、补充咖啡、**补充黄油**等所有在 `AUTO_REPLENISH` 中的补充项）。
- 适用：材料已足够、无需在游戏内点补充键的场景。

### 3. 无猫模式（`no_cat`）

- 与普通模式相同：主配料 + **全部补充点击**都会执行。
- **额外规则**：每一次「补充*」点击完成后，**固定再等待 3 秒**（`NO_CAT_REPLENISH_WAIT_SEC`，可中断），再给后续逻辑执行；用于避开猫或其它动画占屏导致的误点。
- 主配料单次点击后若无补充映射，则不会触发这 3 秒等待。

## 配置与扩展

- **识别用图**：`click_shots/`（主配料在 ROI 内匹配；`提交1` 在全图匹配；可选 `锚点1` 仅在关闭窗口左上角锚点模式时用于算偏移）。
- **点击用坐标**：`calib/points.json` 用于**主配料与各「补充*」**槽位；主配料列表见 `ACTIVE_POINT_NAMES`；主配料 → 补充名见 `AUTO_REPLENISH`；GUI 可对「水果4 / 补充水果4」做紫色水果整体像素微调（`purple_fruit_gui_adjust`）。
- **模式常量**：`RUN_MODE_NORMAL`、`RUN_MODE_INFINITE_MATERIAL`、`RUN_MODE_NO_CAT`（见 `gf2_bot.py`）。
- **校验脚本**：`shot_point_mapping.py` 可核对模板文件名与 JSON 点位是否一致。
