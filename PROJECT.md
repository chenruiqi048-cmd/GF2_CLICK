# GF2 点击助手 — 项目说明

面向《少女前线 2：追放》咖啡厅订单的**外部辅助**：在屏幕截图中识别订单区配料图标，再模拟鼠标点击对应位置与提交按钮。不读取、不修改游戏进程内存，不注入 DLL。

---

## 项目概括

| 项目 | 说明 |
|------|------|
| **用途** | 自动识别右上角订单区域内的配料模板，按从左到右优先级点击，并在适当时机点击「提交」。 |
| **识别** | OpenCV 模板匹配（`click_shots/` 中 PNG 与 `calib/points.json` 中标定名一一对应）。 |
| **坐标** | 以锚点与窗口偏移换算点击位置；可选 GUI「锚点修正」「对齐客户区」与标定分辨率一致。 |
| **入口** | `gf2_gui.py`（GUI）、`gf2_bot.py`（命令行）；核心逻辑在 `gf2_bot.py`。 |
| **权限** | 游戏若以管理员运行，本工具也需管理员权限才能向该窗口投递点击（打包 exe 可带 UAC 提权）。 |

**对齐客户区**：通过 Windows 窗口 API 调整游戏窗口**客户区**尺寸，与标定所用分辨率对齐；属于系统层窗口几何操作，**不是**改内存、也不是改游戏内渲染设置。

## 运行模式（GUI 下拉框）

三种模式由 `gf2_bot.run_bot(..., run_mode=...)` 控制，默认 **普通模式**。

### 1. 普通模式（`normal`）

- 在订单 ROI 内识别到某个**主配料**模板（如水果、饮料、黄油等）后，先点击该配料的标定坐标。
- 若该配料在配置里配置了「自动补充」映射（`AUTO_REPLENISH`），则**再点击一次**对应的「补充*」按钮坐标（如「补充水果1」「补充黄油1」等）。
- 一段时间无新识别后，若匹配到「提交」模板，则点击提交。
- 行为即：主配料点击 + **全部适用的补充点击** + 自动提交逻辑（与历史版本一致）。

### 2. 无限材料模式（`infinite_material`）

- 与普通模式相同：仍会识别并点击**主配料**、仍会尝试**自动提交**。
- **区别**：命中主配料后，**一律不再点击**任何「补充*」按钮（包含补充水果、补充饮料、补充咖啡、**补充黄油**等所有在 `AUTO_REPLENISH` 中的补充项）。
- 适用：材料已足够、无需在游戏内点补充键的场景。

### 3. 无猫模式（`no_cat`）

- 与普通模式相同：主配料 + **全部补充点击**都会执行。
- **额外规则**：每一次「补充*」点击完成后，**固定再等待 3 秒**（`NO_CAT_REPLENISH_WAIT_SEC`，可中断），再给后续逻辑执行；用于避开猫或其它动画占屏导致的误点。
- 主配料单次点击后若无补充映射，则不会触发这 3 秒等待。

## 配置与扩展

- **配料与补充**：主配料列表见 `ACTIVE_POINT_NAMES`；主配料 → 补充名见 `gf2_bot.AUTO_REPLENISH`；坐标在 `calib/points.json`。
- **模式常量**：`RUN_MODE_NORMAL`、`RUN_MODE_INFINITE_MATERIAL`、`RUN_MODE_NO_CAT`（见 `gf2_bot.py`）。
- **校验脚本**：`shot_point_mapping.py` 可核对模板文件名与 JSON 点位是否一致。
